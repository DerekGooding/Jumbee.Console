## Remarks

<p>
Up/Down move the highlight (skipping separators and disabled items), Enter/Space choose, Escape or a click outside
dismiss. An item with a <xref href="Jumbee.Console.MenuItem.Submenu" data-throw-if-not-resolved="false"></xref> opens a child menu to its right (Right/Enter or hover to
open, Left to go back); the whole open chain is drawn as this one popup, so submenus nest to any depth. Choosing a
leaf item runs its <xref href="Jumbee.Console.MenuItem.Action" data-throw-if-not-resolved="false"></xref> and raises <xref href="Jumbee.Console.ContextMenu.ItemActivated" data-throw-if-not-resolved="false"></xref>; <xref href="Jumbee.Console.ContextMenu.Closed" data-throw-if-not-resolved="false"></xref> fires
whenever the menu closes.
</p>
<p>
Because the <xref href="Jumbee.Console.Overlay" data-throw-if-not-resolved="false"></xref> hosts a single popup, the menu draws the entire open submenu chain itself: it is
a plain <xref href="Jumbee.Console.Control" data-throw-if-not-resolved="false"></xref> that writes each level's bordered box directly into its buffer (leaving the gaps
between boxes transparent) and routes input to the deepest open level.
</p>

### <a id="Jumbee_Console_ContextMenu_IntrinsicWidth"></a> IntrinsicWidth\(\)

An intrinsic, fixed width in cells this control always wants regardless of the space its parent offers, or
0 (the default) to fill the parent's width.

```csharp
protected override int IntrinsicWidth()
```

#### Returns

 int

#### Remarks

Unlike <xref href="Jumbee.Console.Control.MeasureHeight(System.Int32)" data-throw-if-not-resolved="false"></xref> — a content height honored only when the parent is unbounded — an
intrinsic size is authoritative even under a finite parent. Override on adornment controls with a genuine
fixed extent (e.g. a vertical <xref href="Jumbee.Console.TextLabel" data-throw-if-not-resolved="false"></xref>, one column wide) so a docking/layout parent can't
stretch them to fill the region.
